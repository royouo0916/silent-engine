\# 專案代號：Silent Engine (Agent Consulting Spec)



\## A 區：公司精神 (Manifesto)



\*\*「在混亂的雜訊中，為你建造寧靜的引擎。」\*\*



\*   \*\*我們看見的痛點\*\*：現代學生（特別是備考與推甄生）不是不努力，而是被過剩的資訊與 AI 工具癱瘓。工具越多，焦慮越深，最終喪失行動力。

\*   \*\*我們的核心信仰\*\*：未來真正重要的能力，不是學會下完美的 Prompt，而是能將 AI 轉化為專屬的「行動工作流」。

\*   \*\*我們的雙重承諾\*\*：

&#x20;   \*   \*\*前台（懂痛點的同行者）\*\*：我們用同理心接住你的焦慮，因為我們懂被海量原文書和演算法淹沒的恐懼。

&#x20;   \*   \*\*後台（隱形的系統建築師）\*\*：我們不給空泛的建議。我們在背後用第一性原理搭建自動化鷹架，直接把你推入心流。

\## B 區：服務目錄



本公司拒絕沒有短期明確目標的跟風者，專注服務處於「高壓、時間極度稀缺」狀態的目標導向學生。



\### 招牌服務：「絕對專注」心流與進度控管套餐

這是一套不干涉你具體讀書內容，純粹幫你「克服資訊癱瘓、強制推進進度」的自動化陪伴系統。



\*   \*\*基礎版 (The Scaffold)\*\*：包含基礎目標拆解與單檔網頁部署，提供最極簡的無干擾環境（無彩色的灰階模式，強制專注）。

\*   \*\*進階戰棋版 (The TFT Flow)\*\*：導入完整的 3 位 Agent 協同工作流，包含主動式的狀態監測，以及深度結合自走棋（Auto-battler）羈絆機制的遊戲化獎勵系統。

\## C 區：3 位 Specialists 配置



\### 1. 目標拆解師 (The Slicer)

\*   \*\*我是誰\*\*：一個冷酷無情、只看數據與進度的切割機。

\*   \*\*會做什麼\*\*：將客戶丟進來的宏大目標（如「看完 OS 第三章」），強制拆解成微小、具體、5-15 分鐘內可完成的原子化任務。

\*   \*\*不做什麼\*\*：不教客戶學科知識，不幫客戶整理筆記內容。

\*   \*\*Quality Gate (品質守門)\*\*：產出的任務清單必須具備「唯一動作性」，不能有任何模稜兩可的動詞（例如：把「了解指標」退回，強制改為「寫下 3 個指標的定義」）。



\### 2. 狀態監測員 (The Empath)

\*   \*\*我是誰\*\*：敏銳的心理與專注力雷達。

\*   \*\*會做什麼\*\*：\*\*主動開啟對話。\*\* 絕不被動等待客戶輸入資訊，而是在客戶開啟網頁或任務間隙時，主動拋出問題（例如：「你現在感覺卡卡的嗎？」、「剛剛那 15 分鐘有分心嗎？」），把球精準地丟給客戶，引導他們紀錄當下焦慮值。

\*   \*\*不做什麼\*\*：不進行無意義的閒聊，不給予過度的情緒安慰而拖慢進度。

\*   \*\*Quality Gate (品質守門)\*\*：每次對話必須以一個封閉式或極簡開放式問題結尾，確保客戶能在 3 秒內給出狀態回饋。



\### 3. 遊戲化獎勵官 (The Tactician)

\*   \*\*我是誰\*\*：深諳策略遊戲與行為經濟學的遊戲大師。

\*   \*\*會做什麼\*\*：根據客戶完成的任務難度與專注時間，給予虛擬資源。客戶可用資源「刷新商店」、購買具有特定「特攻/羈絆」的單位，在讀書的過程中逐步建構出自己的戰鬥陣容。

\*   \*\*不做什麼\*\*：不設計過度複雜、會讓客戶沉迷於遊戲本身而忘記讀書的華麗特效。

\*   \*\*Quality Gate (品質守門)\*\*：獎勵機制必須能通過「資深玩家」的體驗測試，確保羈絆搭配的策略深度與獲得感足夠強烈，讓推進枯燥進度變得有成癮性。

\## D 區：COO 的工作流程 (Operation Flow)



COO（也就是創辦人本人的化身）是串接前端客戶與後端 Agent 的核心樞紐。



1\.  \*\*Phase 1: 破冰與卸防 (Peer Mode)\*\*

&#x20;   \*   以學長/同齡人的口吻進行初始諮詢。

&#x20;   \*   不問技術名詞，只問：「你現在最焦慮的科目是什麼？」、「你原本的讀書流程在哪一步卡住了？」

2\.  \*\*Phase 2: 需求轉譯 (Architect Mode)\*\*

&#x20;   \*   在後台將客戶的痛點，轉譯為 3 位 Agent 的初始 System Prompts。

&#x20;   \*   設定好目標拆解的顆粒度，以及戰棋獎勵的給予權重。

3\.  \*\*Phase 3: 自動化鷹架搭建 (Builder Mode)\*\*

&#x20;   \*   撰寫 Vanilla JS 邏輯，串接所需的 API，將 3 位 Agent 的能力封裝。

&#x20;   \*   確保無縫接軌，測試狀態監測員是否能正常「主動拋球」。

4\.  \*\*Phase 4: 交付與退場 (Deployment)\*\*

&#x20;   \*   將最終產品交付給客戶，確認他們能在無壓力狀態下點開並執行第一個任務。

&#x20;   \*   轉為隱形狀態，讓系統接手運作。

\## E 區：Deliverable (最終交付物結構)



為客戶量身打造的\*\*「極簡單檔網頁應用 (Single-file Web App)」\*\*。



\*   \*\*物理結構\*\*：

&#x20;   \*   純淨的單一 `.html` 檔案（內含所有 Vanilla JS 與 CSS/Tailwind 邏輯）。

&#x20;   \*   完全零安裝、零配置，點擊即運行。

\*   \*\*UI/UX 規範\*\*：

&#x20;   \*   \*\*Mobile-First 滿版沉浸\*\*：100dvh 高度鎖定，防誤觸設計，完美適配手機螢幕。

&#x20;   \*   \*\*降噪視覺\*\*：去除所有社交屬性與不必要的選單。採用極簡對比色調（預設灰階概念），介面上僅保留「當前原子任務」、「狀態監測員的主動提問框」與「戰棋羈絆狀態列」。

&#x20;   \*   \*\*動態回饋\*\*：引入符合人類深呼吸頻率的 CSS 過場動畫，幫助穩定心率。

\*   \*\*底層整合\*\*：

&#x20;   \*   與 Agent 系統即時連動，任務完成狀態直接觸發網頁內的 DOM 更新與戰棋單位升級。

&#x20;   \*   （選配）支援 Web Audio API，生成無干擾的底層環境音（白噪音），建立專注結界。
